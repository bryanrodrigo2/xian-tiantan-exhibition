# 数字博物馆项目工作流与交互逻辑说明

## 一、 前期数据处理与建模

### 1. 大疆倾斜摄影处理
本项目采用无人机倾斜摄影技术进行遗址实景采集，摒弃了传统的摄影测量软件，全面采用大疆生态工作流以确保数据精度与处理效率。

*   **大疆智图 (DJI Terra) 三维重建**：将无人机采集的多角度高清影像数据导入大疆智图软件。利用其强大的空三解算能力，进行自动化的特征点匹配与密集点云生成，最终输出高精度的实景三维模型（通常为 OBJ 或 PLY 格式）。
*   **大疆智模 (DJI Modify) 模型修复**：由于实景扫描模型常伴有破损、噪点或结构缺失，需将初步生成的模型导入大疆智模。在此软件中进行网格拓扑优化、破损孔洞修补、悬浮物去除以及纹理色彩修复，最终导出一个干净、结构完整的底模文件。

### 2. 3ds Max 三维建模
大疆智模输出的 OBJ 文件虽然外观完整，但其网格布线通常较为杂乱，不适合直接用于游戏引擎的实时渲染。因此需要进行专业的“翻模”处理。

*   **底图翻模与重新拓扑**：将修复好的 OBJ 模型导入 3ds Max 作为底层参考（底图）。建模师依据底图的几何形态，使用规范的四边形网格（Quad Topology）重新绘制天坛的圆形祭坛、台阶、栏杆及壝墙等核心构件。这一步骤极大地优化了模型面数，确保了在 UE5 中的运行帧率。
*   **场景扩建与完善**：在核心遗址模型翻模完成后，继续在 3ds Max 中创建配套的周边环境模型。这包括具有现代极简风格的“数字展厅”、还原历史风貌的“遗址公园”景观，以及供玩家停留的“休息区”设施。所有模型在导出前均完成 UV 展开与材质 ID 分配。

## 二、 UE5 引擎核心交互系统

本章节详细阐述 UE5 蓝图（Blueprint）中各项核心交互功能的底层逻辑与工作流程。

### 1. AI 智能讲解员系统
本项目通过接入大语言模型，实现玩家与“大唐守坛人”NPC 的自然语言交互。

*   **工作流程**：
    1.  **触发对话**：玩家靠近 NPC，按下交互键激活对话 UI。
    2.  **数据封装**：玩家在 UI 输入框中输入文字（或通过语音转文字），蓝图将玩家输入的文本与预设的 System Prompt（如：“你现在是唐代天坛的守坛人...”）封装为 JSON 格式。
    3.  **API 请求**：使用 UE5 的 **VaRest 插件**（或内置的 HTTP 模块），向 DeepSeek API 发送 POST 请求。
    4.  **解析与播报**：蓝图接收到 DeepSeek 返回的 JSON 响应后，提取其中的文本内容，显示在对话 UI 上，并同步调用 TTS（Text-to-Speech）插件进行语音播报。

### 2. 展厅四大核心功能工作流

#### ① 物品交互系统
用于玩家近距离观察展厅内的文物或模型细节。

*   **工作流程**：
    1.  **碰撞检测**：玩家角色（Character）走进包裹着展品的**触发盒（Box Collision）**。
    2.  **UI 提示**：触发盒的 `OnComponentBeginOverlap` 事件被激活，在屏幕中央显示“按 [E] 键查看详情”的提示 UI。
    3.  **按键响应**：玩家按下 E 键，触发交互事件。
    4.  **控制权交接**：蓝图调用 `Disable Movement` 冻结玩家角色的移动与视角旋转；同时调用 `Set Input Mode UI Only`，将鼠标指针显示出来，交由 UI 面板控制。
    5.  **展示详情**：在屏幕上实例化并显示（Add to Viewport）该展品的详情 Widget UI。UI 中包含图文介绍，以及一个通过 Scene Capture 2D 渲染的、可使用鼠标拖拽旋转的 3D 模型特写。
    6.  **退出交互**：玩家点击 UI 上的“关闭”按钮或再次按 E 键，销毁该 Widget，调用 `Set Input Mode Game Only` 隐藏鼠标，并恢复玩家的移动能力。

#### ② 视频播放系统
用于在展厅的大屏幕上播放历史纪录片或复原动画。

*   **工作流程**：
    1.  **碰撞检测**：玩家进入大屏幕前方的触发盒区域。
    2.  **UI 提示**：屏幕提示“按 [E] 键播放视频”。
    3.  **触发播放**：玩家按下 E 键，蓝图获取场景中的 **Media Player** 组件引用。
    4.  **加载媒体**：调用 Media Player 的 `Open Source` 节点，加载指定的视频文件（MP4）。视频画面将通过 Media Texture 实时投射到大屏幕的材质上。
    5.  **停止播放**：玩家再次按下 E 键，或走出触发盒区域（触发 `OnComponentEndOverlap` 事件），蓝图调用 Media Player 的 `Pause` 或 `Close` 节点，视频停止播放，屏幕恢复默认待机画面。

#### ③ 登录系统
确保用户身份的唯一性，用于后续的成绩记录与进度保存。

*   **工作流程**：
    1.  **游戏初始化**：在 `GameMode` 的 `BeginPlay` 事件中，首先调用 `Disable Movement` 冻结玩家，并 `Set Input Mode UI Only` 显示鼠标。
    2.  **显示登录 UI**：实例化全屏的“登录/注册 Widget”并添加到屏幕。背景可设置为模糊的展厅场景。
    3.  **信息输入**：玩家在 Editable Text 框中输入账号和密码，点击“登录”按钮。
    4.  **本地校验（简化版）**：蓝图读取玩家输入，与预先存储在 `GameInstance` 或本地 SaveGame 文件中的账号密码数据进行字符串比对（`Equal (String)` 节点）。
    5.  **校验结果处理**：
        *   **失败**：在 UI 上显示红色的“账号或密码错误”提示文本。
        *   **成功**：调用 `Remove From Parent` 销毁登录 UI；调用 `Set Input Mode Game Only` 隐藏鼠标；解除玩家移动限制（`Set Movement Mode: Walking`）；将当前登录的用户名保存到 `GameInstance` 的全局变量中，以便全局调用。

#### ④ 答题系统
寓教于乐的互动环节，检验玩家对天坛历史的了解。

*   **工作流程**：
    1.  **触发答题**：玩家与展厅内的“答题终端”模型交互（按 E 键）。
    2.  **UI 弹出与控制交接**：冻结玩家移动，显示鼠标，弹出“答题 Widget”。
    3.  **题目加载**：Widget 初始化时，从预设的结构体数组（包含题目、选项 A/B/C/D、正确答案索引、解析）中随机抽取或按顺序加载第一题，更新 UI 文本。
    4.  **玩家作答**：玩家点击某个选项按钮。
    5.  **逻辑判断**：蓝图获取玩家点击的按钮索引，与当前题目的“正确答案索引”进行比对。
        *   **正确**：按钮背景变绿，播放正确音效，`GameInstance` 中的“当前得分”变量 +10。
        *   **错误**：按钮背景变红，正确选项闪烁提示，播放错误音效。
    6.  **显示解析与切题**：在 UI 下方显示该题的历史解析文本。玩家点击“下一题”按钮，重复步骤 3-5。
    7.  **结算与退出**：所有题目答完后，UI 切换至“结算面板”，显示总得分与评价。玩家点击“结束答题”，销毁 UI，恢复玩家移动控制权。

## 三、 网页端部署与流式传输（Pixel Streaming）

为了让用户无需下载几十 GB 的游戏客户端，直接在浏览器中就能体验高画质的 3D 博物馆，本项目采用了 Epic Games 官方的 **像素流送（Pixel Streaming）** 技术。

**通俗原理解析：**
简单来说，像素流送就像是“云游戏”或“看互动直播”。
1.  **云端运行**：我们将打包好的 UE5 博物馆程序（.exe）放在一台配置了高性能显卡（GPU）的云服务器上运行。
2.  **画面传输**：服务器上的 UE5 引擎会把渲染出来的每一帧精美画面，像视频直播一样，通过网络实时发送给玩家的浏览器。
3.  **指令回传**：当玩家在网页上按下键盘（WASD 移动）或滑动鼠标（旋转视角）时，网页会瞬间把这些操作指令发送回云服务器。服务器上的 UE5 接收到指令后控制角色移动，并立刻把新的画面传回网页。

**前端网页嵌入流程：**
在我的个人网站（基于 React/Vue 构建）中，我创建了一个专门的“交互复原”页面。在这个页面里，我使用了一个 HTML 的 `<iframe>` 标签，就像在网页里挖了一个“窗口”。
我将这个窗口的 `src` 属性指向了云服务器的流送地址（例如：`http://云服务器IP:9000`），并设置了 `allow="fullscreen; microphone"` 权限。这样，玩家只要打开我的网站，点击“启动交互”按钮，这个 iframe 窗口就会无缝接入云端的 UE5 画面。玩家在网页里就能获得媲美本地 3A 游戏的高清沉浸式漫游体验。
