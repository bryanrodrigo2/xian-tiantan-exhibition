# 西安隋唐天坛全息影像设计与交互实现网站介绍

本项目是一个以“西安隋唐天坛”为主题的数字化文化遗产展示平台。网站通过现代前端技术、3D 渲染与 AI 交互，将沉睡的遗址转化为可观、可感、可互动的数字空间。以下是网站核心功能模块及其技术实现的详细介绍。

## 1. 核心功能模块

网站整体分为五个主要功能模块，分别从历史、现状、复原、交互等多个维度展示天坛的文化内涵。

### 1.1 首页 (Home)
首页是整个网站的视觉中心与导航枢纽。
- **视觉设计**：采用深色背景，中心为带有呼吸灯效果和旋转光环的天坛 Logo，营造出神圣而神秘的历史氛围。
- **导航系统**：四个角落分布着通往主要功能区的悬浮按钮（历史、现状、蓝图、交互）。按钮采用磨砂玻璃质感，悬停时有发光和线条动画，并伴有钟鼓音效（悬停钟声，点击鼓声）。
- **开场动画**：首次进入网站时，会播放一段“全息数据资源加载”的开场动画（SplashScreen），增强沉浸感。

### 1.2 历史溯源 (History)
该模块主要梳理天坛的历史沿革。
- **时间轴展示**：以垂直时间轴的形式，详细记录了从隋文帝初建圆丘（582年）到唐睿宗景云祭天（712年）等历代皇帝的祭祀大事件。
- **文献考证**：每个时间节点都配有《大唐郊祀录》、《旧唐书》等古籍文献的详细记载，确保历史信息的严谨性。

### 1.3 现状 (Survey)
展示天坛遗址的当前状态与测绘数据。
- **设计理念**：通过引用学术文献，阐述了“场所精神的数字重构”和“从君权神授到全民共享”的数字化保护理念。
- **CAD 图纸展示**：提供了一个可交互的图纸画廊，包含天坛及周边地区总平图、展厅平面图、立面图、景观平面图等 9 张详细的工程测绘图纸。用户可以点击放大查看细节。
- **现状模型 3D 查看器**：在现状模型标签页中，嵌入了**大疆 DJI 实景三维模型查看器**。用户可以直接在网页中自由旋转、缩放、平移，全方位查看基于无人机倾斜摄影技术生成的遗址高精度三维模型。

### 1.4 蓝图规划 (Blueprint)
展示天坛的复原推导过程与最终效果。
- **复原依据**：分为“古籍考证”和“考古数据”两部分，展示了文献推导图和考古发掘平面图。
- **复原成果**：展示了唐代圜丘的完整复原图，以及保护工程的渲染效果图。所有图片均支持点击全屏放大（Lightbox 效果）。

### 1.5 交互与复原 (Interaction)
这是网站最具特色的核心模块，提供了多种沉浸式体验方式。
- **虚拟展馆体验**：通过集成 UE5 像素流送技术，用户可以直接在网页端进入高精度的 3D 虚拟展馆，自由漫游并探索天坛的复原场景。
- **AI 导览员 (AI Guide)**：集成虚拟数字人导览功能，同样基于像素流送技术，用户可以与 AI 导览员进行互动。
- **手势交互 (Gesture Interaction)**：利用摄像头捕捉用户手势，实现对天坛 3D 模型的控制（如张开手掌粒子消散，握拳粒子聚合，移动手掌旋转模型）。
- **全景对比 (Comparison)**：提供唐代复原场景与现代遗址现状的滑动对比功能，直观感受历史变迁。

## 2. 技术实现方案

网站采用了现代化的前端技术栈，结合 3D 渲染和机器学习算法，实现了高性能的纯前端展示与交互。

### 2.1 基础框架与构建
| 技术项 | 选型 | 说明 |
| :--- | :--- | :--- |
| **核心框架** | React 19 | 使用最新的函数组件和 Hooks 模式构建用户界面。 |
| **构建工具** | Vite | 提供极速的本地开发热更新和优化的生产环境构建。 |
| **开发语言** | TypeScript | 提供严格的类型检查，保证代码的健壮性和可维护性。 |
| **路由管理** | Wouter | 使用轻量级的 Hash 路由模式，完美适配纯静态文件部署。 |

### 2.2 视觉与动画
- **样式方案**：采用 **Tailwind CSS**，通过原子化类名快速实现响应式布局和复杂的 UI 效果（如磨砂玻璃、发光边框）。
- **动画引擎**：广泛使用 **Framer Motion**。从首页的元素入场、Logo 的呼吸脉冲，到页面切换的平滑过渡，均由该库驱动，极大提升了视觉流畅度。
- **UI 组件库**：基于 **shadcn/ui**（如 Card、Tabs、ScrollArea 等），保证了界面风格的统一和高可用性。

### 2.3 3D 渲染与手势交互
这是本项目技术含量最高的部分，完全在浏览器端（客户端）运行，无需后端服务器参与计算。
- **3D 渲染引擎**：使用 **Three.js**。在手势交互页面中，通过 `OBJLoader` 加载天坛的 3D 模型，并将其顶点数据转换为**粒子系统**（`THREE.Points`），从而实现模型消散与聚合的视觉特效。
- **手势识别算法**：集成 Google 的 **MediaPipe Hands** 机器学习模型。通过调用用户摄像头，实时检测 21 个手部关键点，判断用户是“张开手掌”还是“握拳”，并将状态传递给 Three.js 控制粒子动画。
- **模型云端托管**：为了解决 3D 模型文件过大导致网页加载缓慢的问题，将 `.obj` 和 `.mtl` 模型文件托管在**阿里云 OSS（对象存储服务）**上，通过 CDN 加速异步加载。

### 2.4 像素流送 (Pixel Streaming) 与云端托管
在“虚拟展馆”和“AI 导览员”场景中，采用了 Unreal Engine (UE5) 的像素流送技术。
- **实现原理**：高精度的 3D 场景和数字人由高性能 GPU 机器（运行 UE5）实时渲染，然后将画面作为视频流通过 WebRTC 传输到网页端的 `iframe` 中。
- **点量云流托管**：为了保证流送的稳定性和低延迟，本项目的像素流送服务**托管在第三方平台“点量云流”上**。网页端通过配置点量云流提供的信令服务器地址（如 `http://127.0.0.1:9000/s?loadType=auto&group=...`）建立连接。
- **前端集成**：前端代码拦截了浏览器的默认键盘/鼠标事件（如空格、方向键），确保用户的漫游操作指令能实时、准确地传递给云端的 UE 引擎。

### 2.5 部署架构
- **前端托管**：网站的静态资源（HTML/CSS/JS/图片）部署在 **Vercel** 平台上，享受全球 CDN 加速，并绑定了自定义域名 `bryanfinal.xyz`。
- **无后端设计**：网站本身是一个纯前端的单页应用（SPA），所有文本数据（如历史时间轴、图纸信息）均硬编码在本地 JSON 或组件中，**没有使用传统的后端数据库**，极大地降低了维护成本并提升了访问速度。

---

**总结**：该网站巧妙地融合了 React 现代前端开发、Three.js 实时渲染、MediaPipe 边缘计算以及基于点量云流的 UE5 像素流送技术，在无后端数据库的轻量级架构下，打造了一个兼具学术严谨性与前沿科技感的文化遗产数字化展示平台。
